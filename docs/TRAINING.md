# Обучение ML-модели AutoScoutBot

## 1. Локальная модель и Railway

**Да, модель, обученная локально, будет работать на Railway** — если вы положите обученные файлы в репозиторий и сделаете push.

- Модели лежат в каталоге `scoring/models/` (подкаталоги `overall`, `tech_maturity`, `innovation` и т.д. с файлами `.joblib`).
- При деплое Railway клонирует ваш репозиторий и запускает бота; при старте загружаются модели из `scoring/models/` (см. `railway_start.py` → `check_ml_models()`).
- **Что делать после обучения локально:**
  1. Обучите модель (см. ниже).
  2. Закоммитьте и запушьте каталог с моделями:
     ```bash
     git add scoring/models/
     git commit -m "Обновлены ML-модели (обучение на Сколково)"
     git push
     ```
  3. В Railway произойдёт пересборка/рестарт — новый деплой будет использовать уже обновлённые модели.

Если `scoring/models/` не коммитить (например, он в .gitignore), то на Railway моделей не будет и бот будет использовать эвристический скоринг. В вашем проекте `scoring/models/` **не** в .gitignore — значит, коммитить и пушить можно и нужно.

## 2. На каких данных обучается модель

| Этап | Данные | Нужен ли парсинг? |
|------|--------|-------------------|
| **Базовое обучение** | Только **SkolkovoStartups.csv** (база Сколково) | **Нет.** CSV уже есть, парсеры не используются. |
| **Дообучение (retrain)** | Сколково + внешние стартапы (из БД `external_startups` или из `external_batch.jsonl`) | Внешние данные собираются парсерами (ЕГРЮЛ, БФО, новости и т.д.). Сейчас БФО часто даёт 403, ЕГРЮЛ не всегда отдаёт данные — из-за этого «уверенных» внешних записей мало и дообучение может не добавлять ни одной. |

Итого: **основная модель обучается только на Сколково**, по одному CSV. Парсеры нужны только для опционального дообучения на внешних компаниях.

## 3. Как обучать (пошагово)

Всё делается в **терминале** из корня проекта, без кнопок в боте.

### 3.1 Убедиться, что есть CSV

В корне проекта должен быть файл **SkolkovoStartups.csv** (тот же, что использует бот и что указан в `config` / `SKOLKOVO_DATABASE_PATH`).

### 3.2 Обучить модель на Сколково (без внешних данных)

Один из вариантов:

**Вариант A — полный пайплайн (разметка + обучение всех 6 моделей):**

```bash
# Из корня проекта, с активированным venv
python run_train.py
```

Скрипт:
1. Строит proxy-метки по CSV (`scoring/labeled_startups.csv`).
2. Обучает все 6 моделей (overall, tech_maturity, innovation, market_potential, team_readiness, financial_health) и сохраняет их в `scoring/models/`.

**Вариант B — только обучение (если разметка уже есть):**

```bash
python -m scoring.train
```

По умолчанию читается `SkolkovoStartups.csv` из корня. Своий CSV можно задать так:

```bash
python -m scoring.train --csv SkolkovoStartups.csv --model-dir scoring/models
```

**Проверить сохранённые модели (без переобучения):**

```bash
python -m scoring.train --evaluate
```

Выведет R², MAE по каждой целевой переменной по тем же данным.

### 3.3 Дообучение на внешних данных (опционально)

Имеет смысл, когда есть внешние стартапы с нормально заполненными полями (выручка, год, статус и т.д.):

- Либо в БД: таблица `external_startups` (заполняется при использовании «Проверить стартап» в боте).
- Либо в файле: `external_batch.jsonl` после `python scripts/batch_external_fetch.py`.

Команды:

```bash
# Из БД (если есть записи в external_startups)
python -m scoring.retrain

# Из JSONL (после batch_external_fetch.py)
python -m scoring.retrain --jsonl external_batch.jsonl
```

Сейчас из-за того, что парсеры почти не отдают данных (БФО 403, ЕГРЮЛ пусто), по JSONL получается 0 «уверенных» псевдо-меток — дообучение не добавляет примеры и завершается с сообщением «Дообучение отложено». Это нормально: **базовая модель при этом уже обучена на Сколково** и работает; дообучение начнёт что-то добавлять, когда появятся нормальные внешние данные.

## 4. Краткая шпаргалка

| Задача | Команда |
|--------|---------|
| Обучить всё с нуля (разметка + 6 моделей) | `python run_train.py` |
| Только обучить по текущему CSV | `python -m scoring.train` |
| Проверить метрики сохранённых моделей | `python -m scoring.train --evaluate` |
| Дообучить на внешних из БД | `python -m scoring.retrain` |
| Дообучить на внешних из JSONL | `python -m scoring.retrain --jsonl external_batch.jsonl` |
| После обучения — обновить Railway | `git add scoring/models/` → `git commit` → `git push` |

Данные для обучения: **только SkolkovoStartups.csv**. Парсеры для обучения не обязательны; они нужны только для дообучения, и пока они не отдают нормальные данные, дообучение просто не добавляет примеры.
