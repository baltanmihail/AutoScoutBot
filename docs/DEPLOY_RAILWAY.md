# Деплой AutoScoutBot на Railway

## Подготовка репозитория

1. **Инициализация Git** (если ещё не сделано):
  ```bash
   git init
   git add .
   git commit -m "Деплой 5.2"
  ```
2. **Убедитесь, что в репозиторий не попадают:**
  - `config.py` с секретами (он в .gitignore)
  - `.env` (в .gitignore)
  - `*.db`, `rag_index_gigachat.json`, `SkolkovoStartups.csv` — при необходимости добавьте в .gitignore или храните CSV в репо, если он не секретный
3. **Пуш в GitHub:**
  ```bash
   git remote add origin https://github.com/baltanmihail@gmail.com/AutoScoutBot.git
   git branch -M main
   git push -u origin main
  ```

## Настройка Railway

1. Зайдите на [railway.app](https://railway.app), войдите через GitHub.
2. **New Project** → **Deploy from GitHub repo** → выберите репозиторий AutoScoutBot.
3. **Variables** (переменные окружения) — обязательно задайте:
  - `TELEGRAM_TOKEN` — токен бота от @BotFather
  - `GIGACHAT_API_TOKEN` — токен GigaChat API (эмбеддинги + _internal)
  - `NEUROAPI_KEY` — API-ключ NeuroAPI (пользовательские LLM-модели)
  - `ADMIN_IDS` — ID админов через запятую: `5079636941,1856746424`
   Опционально:
  - `NEUROAPI_BASE_URL` (по умолчанию `https://neuroapi.host/v1`)
  - `LLM_MODEL_STANDARD`, `LLM_MODEL_PREMIUM`, `LLM_MODEL_ULTRA` (переопределение моделей)
  - `SKOLKOVO_DATABASE_PATH` (по умолчанию `SkolkovoStartups.csv`)
  - `RAG_ENABLED` = `true` / `false`
  - `RAG_INDEX_FILE`, `RAG_TOP_K`
4. **Тип сервиса:** Worker (не Web). В Railway при деплое из репо по умолчанию подхватывается **Procfile**: команда `worker: python bot.py` запускает бота как воркер.
5. **Файлы данных:**
  - `SkolkovoStartups.csv` должен быть в репозитории или загружаться при старте (например, из внешнего хранилища).  
  - **ML-модели:** каталог `scoring/models/` с обученными моделями (.joblib) должен быть в репозитории. Если вы обучаете модели локально, после обучения выполните `git add scoring/models/` и `git push` — при следующем деплое Railway подхватит новые модели. Без моделей бот работает с эвристическим скорингом.
  - RAG-индекс при первом запуске может строиться 5–10 минут; для ускорения можно собрать индекс локально и положить `rag_index_gigachat.json` в репо (если не в .gitignore) или на объёмное хранилище Railway.
6. После деплоя смотрите логи в Railway — бот должен вывести что-то вроде «Бот запускается...» и «База Сколково загружена».

## Локальная разработка с .env

Создайте в корне проекта файл `.env` (он не коммитится):

```env
TELEGRAM_TOKEN=ваш_токен_бота
GIGACHAT_API_TOKEN=ваш_токен_gigachat
NEUROAPI_KEY=sk-ваш_ключ_neuroapi
ADMIN_IDS=5079636941,1856746424
```

Не создавайте `config.py` — тогда при запуске `python bot.py` будет использоваться `config_env.py` и переменные из `.env`.

Если хотите по-прежнему использовать локальный `config.py` с секретами — просто оставьте его; тогда он будет подхвачен первым и переменные окружения не нужны.

## Запуск с ПК (локально)

Чтобы запускать бота с компьютера, когда потребуется:

1. **Вариант с config.py**
  Оставьте в корне проекта свой `config.py` (он в .gitignore, в репозиторий не попадёт). Запуск:
   Бот сначала подхватит `config.py`, переменные окружения не нужны.
2. **Вариант без config.py**
  Создайте в корне `.env` с теми же переменными (TELEGRAM_TOKEN, GIGACHAT_API_TOKEN, ADMIN_IDS и т.д.) и запускайте `python bot.py` — настройки возьмутся из `config_env.py` и `.env`.

Один и тот же репозиторий можно и деплоить на Railway (через Variables), и запускать локально (через config.py или .env).

## Интеграция с Backend API (опционально)

Если поднимаете **FastAPI backend** (PostgreSQL + ML-модели), бот может использовать его для поиска и скоринга:

- В **config.py** или **.env** задайте `BACKEND_URL` (например `https://your-app.railway.app` или `http://localhost:8000`).
- При **поиске** бот сначала запрашивает `/search/` у backend; при успехе показывает результаты с ML-ранжированием; при недоступности backend — локальный RAG/фильтры.
- При **глубоком анализе** бот дополняет отчёт данными из `/score/full` (SHAP-факторы, финансы по годам), если backend доступен.

Чтобы бот работал **без backend**, оставьте `BACKEND_URL` пустым в config или не задавайте переменную в Railway.

## Тестирование перед презентацией

### 1. Поиск стартапов (ИИ, запрос текстом)

- Запустить бота → «Анализ с помощью ИИ» или «Анализ с помощью фильтров».
- Ввести текстовый запрос (например: «биотех стартапы», «ИИ в медицине», «проект по computer vision для ритейла»).
- Проверить: приходит список стартапов с карточками, реквизитами, оценками; кнопки «Глубокий анализ» и «Проверить стартап» работают.

### 2. Проверка по ИНН (/check)

- Нажать «Проверить стартап» → ввести ИНН **7714937779** (Фармхаб, участник Сколково).
  - Ожидание: карточка, ML-скоринг (Сколково), при наличии — блок финансов (БФО).
- Ввести ИНН **9727083038** (ООО «ЛОНЖ ГРУПП», внешняя компания).
  - Ожидание: карточка из EGRUL, при наличии отчётности в ФНС — блок «ФИНАНСОВЫЕ ПОКАЗАТЕЛИ (БФО) (ФНС)» и коэффициенты (ликвидность, Z-Альтмана и т.д.); ML-скоринг по внешним данным.

### 3. Глубокий анализ

- После поиска выбрать один стартап → «Глубокий анализ».
- Проверить: отчёт с TRL/IRL/MRL/CRL, финансами, рекомендациями; при настроенном Backend API — в конце блока «ML-скоринг (Backend API)» с SHAP-факторами.

### 4. Backend (если используется)

- Поднять backend: `uvicorn backend.app:app --reload --port 8000`.
- В config задать `BACKEND_URL = "http://localhost:8000"`.
- Повторить поиск и глубокий анализ: в логах бота должно появиться «Поиск выполнен через Backend API».
