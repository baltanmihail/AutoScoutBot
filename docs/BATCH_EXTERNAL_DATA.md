# Пакетный сбор внешних данных и сравнение с эталоном Сколково

## Цель

Собрать по **всем** (или выборке) стартапам из базы Сколково данные из внешних источников (ЕГРЮЛ, БФО ФНС, Checko, MOEX, новости), сохранить в отдельное хранилище, затем:

1. Прогнать нашу ML-модель по тем же стартапам, но с учётом собранных внешних данных (или только по внешним данным).
2. Сравнить предсказания с эталонными метками Сколково.
3. При улучшении метрик — дообучить модель через `scoring/retrain.py`, не угробив текущую (есть пороги R² и откат).

## Шаг 1: Пакетный сбор

Запуск из **корня проекта** (чтобы подхватились `config` и путь к CSV Сколково):

```bash
# Все стартапы с ИНН (долго, с задержкой 1 сек между запросами)
python scripts/batch_external_fetch.py

# Тест на 100 стартапах
python scripts/batch_external_fetch.py --limit 100

# С увеличенной задержкой и своим файлом
python scripts/batch_external_fetch.py --limit 200 --delay 2.0 --output data/external_batch.jsonl
```

Параметры:

| Параметр   | Описание                          | По умолчанию            |
|-----------|-----------------------------------|--------------------------|
| `--limit` | Макс. число стартапов (для теста) | без лимита               |
| `--delay` | Задержка между запросами (сек)   | 1.0                      |
| `--output`| Выходной JSONL-файл               | `external_batch.jsonl`   |

Формат одной строки JSONL:

```json
{"inn": "...", "name": "...", "skolkovo_id": "...", "external": {"bfo": {...}, "egrul": {...}, "news": {...}, ...}}
```

## Шаг 2: Сравнение с эталоном

Дальнейшие варианты:

- **Запуск дообучения по JSONL:** после сбора данных можно запустить дообучение без БД:
  ```bash
  python -m scoring.retrain --jsonl external_batch.jsonl --csv SkolkovoStartups.csv
  python -m scoring.retrain --jsonl external_batch.jsonl --dry-run --min-external 10
  ```
  Нужен файл `SkolkovoStartups.csv` в корне проекта (или путь через `--csv`). Если CSV нет — скрипт выведет сообщение и завершится.

- **Только сравнение:** загрузить эталонные метки Сколково (из `scoring/labeled_startups.csv` или из пайплайна разметки), для каждой записи из `external_batch.jsonl` по `inn`/`skolkovo_id` взять эталонный score, построить фичи из «Сколково + external» (или только из external), прогнать текущую ML-модель, посчитать MAE/R² между предсказанием и эталоном.
- **Дообучение:** использовать собранные внешние данные как доп. источник для `scoring/retrain.py`: либо добавлять в таблицу `external_startups` (как при `/check`), либо загружать JSONL флагом `--jsonl`. Важно: retrain уже проверяет R² на отложенной выборке Сколково и откатывает модель при падении метрик — потенциал не угробим, если не отключать эти проверки.

## Шаг 3: Обучение/дообучение без риска

- Текущая модель обучается на `scoring/labeled_startups.csv` (эталон Сколково).
- Дообучение: `python -m scoring.retrain` — подмешивает внешние стартапы с весами ниже 1.0, на валидации смотрит R² на Сколково; при падении ниже порога — откат.
- Перед продакшеном всегда можно прогнать `python -m scoring.train --evaluate` по сохранённым моделям и убедиться в метриках.

## Что нужно для полного цикла

1. **Запустить пакетный сбор** — уже реализован скрипт выше.
2. **Скрипт сравнения** (опционально): загрузка JSONL + Skolkovo CSV, объединение по ИНН, построение фичей (обогащённых из external), предсказание ML, сравнение с эталоном (MAE, R² по подвыборке с метками).
3. **Интеграция с retrain**: либо экспорт из JSONL в формат, который читает `retrain.py` (список dict с полями для `extract_features`), либо запись в `external_startups` с осторожным весовым режимом, чтобы не перекосить модель в сторону внешних данных.

Рекомендация: сначала собрать выборку (`--limit 200`), вручную прогнать сравнение и retrain на ней, убедиться в метриках, затем запускать полный сбор и плановое дообучение.

**Важно:** для дообучения нужны внешние стартапы с достаточно заполненными признаками (выручка, год, статус из ЕГРЮЛ/БФО). Если в JSONL только данные новостей (БФО даёт 403, ЕГРЮЛ не парсится), псевдо-метки получаются с низкой уверенностью и retrain пропускает шаг («0 уверенных»). Решение: починить/настроить парсеры БФО и ЕГРЮЛ или накапливать записи через `/check` в боте (там сохраняются в БД и подходят для retrain без `--jsonl`).
