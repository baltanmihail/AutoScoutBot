## Текущее состояние

- **Бот:** поиск по базе (ИИ + фильтры), проверка по ИНН (`/check`), балансы запросов (Gemini/Sonnet/Opus), оплата.
- **Данные:** ЕГРЮЛ, БФО (через Checko API, при наличии ключа), новости, MOEX; парсер БФО ФНС (bo.nalog) — при 403 не используется.
- **Отчёт `/check`:** карточка компании, исходные БФО, расчётные коэффициенты (ликвидность, платёжеспособность, рентабельность, Z-Альтман/Таффлер, динамика), ML-скоринг (6 измерений), новости, пояснения по данным.
- **ML:** обучение на базе Сколково, фичи из БФО/EGRUL/Checko, преддиктивная аналитика во внешних проверках и при поиске.

## Приоритеты развития (по шагам)

### 1. Данные и отчётность (сейчас)

- **Checko /company:** ✅ подтягиваются в `fetch_all`, блок «Реквизиты и контакты (Checko)» в отчёте `/check`.
- **Расчёты и риски:** ✅ коэффициенты считаются в `scoring/bfo_ratios.py`, выводятся в отчёте и **сохраняются в БД** в `external_startups.ratios_json` при каждом `/check` (см. `docs/SCHEMA_V2.md`).
- **Честность по данным:** если БФО/финансов нет — в блоке 6 отчёта явно указано «Финансовая отчётность не найдена».

### 2. Конвейер БФО и эталоны

- Устойчивая загрузка БФО через Checko (уже есть для выборки; при полном прогоне — лимиты API и стоимость).
- Пакетный сбор: `scripts/batch_external_fetch.py`, сохранение в JSONL; сравнение с эталоном Сколково, дообучение через `scoring/retrain.py` без порчи метрик (R², откат).

### 3. ML и фичи

- Расширить feature engineering новыми финансовыми и риск-фичами (уже часть плана).
- Переобучение с учётом внешних данных (retrain по JSONL/БД) с проверкой на отложенной выборке.

### 4. Deep-анализ и экспорт

- Глубокий анализ (LLM): 4 фазы, TRL/IRL/MRL/CRL, риски и возможности.
- Экспорт профессионального отчёта в docx по выбранному стартапу.

### 5. Стартапы без БФО

- Конвейер оценки по презентациям/текстам (СберЮнити и др.), когда финансовой отчётности нет.

### 6. Позиционирование

- Сводка по конкурентам (Пеликан, Тамара, СберЮнити и др.) и чёткое позиционирование AutoScoutBot в продуктовой стратегии.

## Как двигаться дальше

1. Включить в `ParserManager` вызов Checko `/company` и вывод этих данных в отчёте `/check`.
2. Прогнать пакетный сбор на выборке, убедиться в качестве данных и стоимости.
3. По мере появления данных — дообучать модель (retrain), расширять фичи и отчёт (графики/таблицы при необходимости).
4. Добавить экспорт docx и конвейер для стартапов без БФО по приоритету продукта.

План обновляется по мере выполнения пунктов.
