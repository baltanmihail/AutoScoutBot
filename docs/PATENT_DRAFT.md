# Заявка на полезную модель / изобретение

## Название

**Система и способ автоматизированной оценки инновационных проектов (стартапов) на основе мультимодальной AI-модели с обучением на экспертных данных и перекрёстной верификацией из открытых источников**

---

## 1. Область техники

Изобретение относится к области информационных технологий, а именно к системам анализа и оценки инновационных предприятий (стартапов) с использованием методов машинного обучения и обработки естественного языка.

---

## 2. Уровень техники

Существующие решения для анализа стартапов делятся на:

1. **Консалтинговые подходы** (Сбер Юнити, PwC, McKinsey) — ручной анализ экспертами, высокая стоимость (от 500 тыс. руб. за проект), субъективность оценок, низкая масштабируемость.

2. **Базы данных стартапов** (Crunchbase, PitchBook, СПАРК-Интерфакс) — агрегация информации без AI-скоринга, отсутствие автоматической оценки качества стартапа.

3. **Каталоги** (Стартапнавигатор Сколково) — списки компаний без аналитики, невозможность ранжирования по качеству.

Недостатки существующих решений:
- Отсутствие автоматической многофакторной AI-оценки стартапов
- Невозможность масштабирования на тысячи компаний в реальном времени
- Отсутствие перекрёстной верификации данных из нескольких источников
- Непрозрачность оценок (отсутствие объяснимости)

---

## 3. Раскрытие изобретения

### 3.1 Техническая задача

Создание автоматизированной системы оценки стартапов, которая:
- Обучается на экспертных оценках (supervised learning)
- Комбинирует табличные, текстовые и категориальные признаки
- Автоматически обогащает данные из открытых государственных и коммерческих источников
- Обеспечивает объяснимость каждой оценки (explainable AI)
- Самосовершенствуется на основе обратной связи пользователей

### 3.2 Существенные признаки

1. **Мультимодальная модель оценки** — ансамбль из:
   - Gradient Boosting (XGBoost/LightGBM) для числовых и категориальных признаков (35 features)
   - Domain-specific embeddings для текстовых описаний (описание компании, продуктов, технологий)
   - Взвешенная комбинация предсказаний для итоговой оценки

2. **35-мерный вектор признаков стартапа:**
   - Финансовые (12): выручка и прибыль за 6 лет (2020-2025)
   - Технологические (8): TRL, IRL, MRL, CRL, агрегаты
   - Инновационные (6): патенты, AI-технологии, разнообразие технологий
   - Категориальные (5): кластер, статус, регион, год, стаж
   - Производные (4): стадия, маржинальность, тренды роста

3. **Proxy-разметка и обучение на экспертных данных:**
   - Начальная разметка через proxy-метрики (TRL/IRL/MRL/CRL + финансы + патенты)
   - Дообучение при получении реальных экспертных оценок
   - 6-компонентная оценка: технологическая зрелость, инновационность, рыночный потенциал, команда, финансы, общий балл

4. **Перекрёстная верификация из открытых источников:**
   - ЕГРЮЛ (ФНС) — юридический статус, учредители, адрес
   - Бухгалтерская отчётность (Checko.ru) — актуальные финансы
   - Новостные агрегаторы — упоминания, репутация
   - Патентная база ФИПС — верификация патентов
   - Каждый источник имеет уровень доверия (source_authority, 0-1)

5. **SHAP-объяснимость (Explainable AI):**
   - Для каждой оценки вычисляются SHAP-значения
   - Пользователь видит: "Стартап получил 8.5/10 потому что: TRL=7 (+1.2), рост выручки 300% (+2.1), патенты (+0.8)"
   - Факторы, снижающие оценку, также раскрываются

6. **Самообучение на обратной связи:**
   - Логирование пользовательских запросов и выбранных результатов
   - Implicit feedback: какие стартапы пользователи изучают подробнее
   - Периодическое дообучение модели на новых данных

### 3.3 Технический результат

- Автоматическая оценка 5000+ стартапов с точностью MAE < 0.5 (шкала 1-10)
- Время оценки одного стартапа: < 100 мс (inference)
- Объяснимость: для каждой оценки предоставляются топ-5 факторов «за» и «против»
- Автоматическое обогащение данных из 4+ внешних источников
- Масштабируемость: серверная архитектура на Railway (PostgreSQL + pgvector + FastAPI)

## 4. Формула изобретения

### Независимый пункт (система)

Система автоматизированной оценки инновационных проектов (стартапов), содержащая:
- модуль извлечения признаков, выполненный с возможностью формирования 35-мерного вектора из финансовых, технологических, инновационных, категориальных и производных признаков стартапа;
- модуль машинного обучения, включающий ансамбль моделей градиентного бустинга и текстовых embeddings, обученных на экспертных оценках не менее 5000 стартапов;
- модуль обогащения данных, выполненный с возможностью автоматического сбора и перекрёстной верификации информации из открытых государственных (ЕГРЮЛ) и коммерческих (Checko.ru) источников с присвоением уровня доверия каждому источнику;
- модуль объяснимости на основе SHAP-анализа, обеспечивающий декомпозицию итоговой оценки на вклады отдельных признаков;
- серверный модуль с REST API и векторной базой данных (pgvector) для семантического поиска по описаниям стартапов.

### Зависимые пункты

2. Система по п. 1, отличающаяся тем, что модуль машинного обучения использует 6-компонентную оценку: технологическая зрелость (TRL, IRL, MRL, CRL), инновационность (патенты, AI), рыночный потенциал (IRL, отрасли), готовность команды (CRL), финансовое здоровье (тренды выручки/прибыли), общий балл (взвешенная комбинация).

3. Система по п. 1, отличающаяся тем, что модуль обогащения данных выполнен с возможностью фоновой работы с регулируемой периодичностью обновления и контролем частоты обращений к внешним источникам (rate limiting).

4. Система по п. 1, отличающаяся тем, что дополнительно содержит модуль самообучения, выполненный с возможностью дообучения модели на основе обратной связи пользователей (implicit + explicit feedback).

### Независимый пункт (способ)

Способ автоматизированной оценки стартапов, включающий:
- этап извлечения 35-мерного вектора признаков из данных стартапа;
- этап предсказания оценок с использованием ансамбля моделей;
- этап обогащения данных из 4+ внешних источников с перекрёстной верификацией;
- этап генерации объяснения с использованием SHAP-анализа;
- этап формирования итогового отчёта с рекомендацией.

## 5. Описание чертежей

### Рис. 1 — Общая архитектура системы

```
[Telegram Bot] -> [FastAPI Server] -> [Scoring Model]
                                   -> [Embeddings (pgvector)]
                                   -> [PostgreSQL]
[Data Parser] -> [ЕГРЮЛ API]
              -> [Checko.ru]
              -> [Google News RSS]
              -> [PostgreSQL]
```

### Рис. 2 — Pipeline оценки стартапа

```
Данные стартапа -> Feature Extraction (35 признаков)
               -> ML Model (LightGBM) -> Score (1-10)
               -> SHAP Explainer -> Объяснение
               -> Report Generator -> Текстовый отчёт
```

### Рис. 3 — Схема обогащения данных

```
Стартап (ИНН) -> [ЕГРЮЛ] -> Статус, учредители (authority=0.95)
              -> [Checko] -> Финансы (authority=0.80)
              -> [News]   -> Упоминания (authority=0.60)
              -> Верификация -> Обновление БД
```

## 6. Реферат

Предложены система и способ автоматизированной оценки инновационных проектов (стартапов) на основе мультимодальной AI-модели. Система включает модуль извлечения 35-мерного вектора признаков, модуль машинного обучения на основе ансамбля gradient boosting и текстовых embeddings, модуль автоматического обогащения данных из открытых государственных и коммерческих источников с перекрёстной верификацией, а также модуль SHAP-объяснимости для декомпозиции оценок. Обучение модели проводится на экспертных оценках 5000+ стартапов Сколково. Технический результат — повышение точности и объективности оценки стартапов при одновременном обеспечении объяснимости и масштабируемости системы.

## 7. Действия для подачи

1. **Обратиться в патентный отдел Сколтеха** — уточнить процедуру, получить помощь с оформлением
2. **Выбрать тип заявки**: полезная модель (быстрее, 6-8 мес.) или изобретение (дольше, но весомее)
3. **Оплатить пошлину**: ~10-15 тыс. руб. (возможна льгота для студентов)
4. **Подать через ФИПС** (Роспатент): https://www.fips.ru/
5. **Срок рассмотрения**: 6-12 месяцев для полезной модели, 12-24 для изобретения

Подача заявки фиксирует приоритет и не мешает дальнейшей разработке.
